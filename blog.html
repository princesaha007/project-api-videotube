Discuss the scope of var, let, and const

var: if we declare a variable with var we can access it globally as var has function scope so after declaring with var it will be hoisted at the top of the function and this can create conflict within the codes as some unexpected scenario can be occurred

let: let is different from var. it is block scoped and we cannot access it globally as long as it is not initialized or defined. they cant cross the boundary of {}. so it is a good practice to use let.

const: const is also block scoped as let and we have to declare that outer {} to access it globally. and like the var and let the values cannot me changed after we initialize the value

Tell us the use cases of null and undefined

Undefined:

  • 1. variable that is not initialized will give undefined (let first ;)
  • 2. function with no return
  • 3. set a value directly to undefined
  • 4. if return has nothing on the right side will return undefined
  • 5. property that doesn't exists on an object
  • 6. accessing array elements outside of the index range
  • 7. deleting an element inside an array
  • 8.parameter that is not passed

Null:

Creating an initial value for a variable or property to show that it presently doesn't point to anything of significance.

What do you mean by REST API?

REST is an architectural set of restrictions rather than a protocol or standard. REST can be implemented by API developers in a variety of ways.
Rest API is used Delivering static resource representations in JSON . It is an interface that allows two computers to communicate securely using the internet.Depending on how the API is designed by the developers, the REST API request and response are different. REST APIs enable clients to request, create, update, and remove data resources via the internet by adhering to statelessness, standard interfaces, and layered systems concepts. They are frequently created in web and mobile apps to facilitate simple and scalable data transmission between various platforms.
